# Piton — українcька транскрипційна мова скриптів 🐍

**Piton** — це компактний інтерпретатор на Go, що дозволяє писати логіку українською, але виконати її, передавши на ходу вхідні вирази латиницею за принципом “як чую, так і пишу”. Мова об'єднує педагогічний синтаксис, інтуїтивні математичні вирази та чисте структурування через відступи.

## Акценти
- 🇺🇦 Переклад англомовних структур на звичні українські слова (`functia`, `nekhay`, `drukuvaty`, `yaksho`, `poky`, `kinets`).
- 🧠 Інструкційна підтримка помилок: парсер завжди повідомляє `Ryadok [N]: Ya tut interpretator...`, що знижує поріг входження.
- 🧮 Вбудовані оператори `stupin`, `korin`, `loh10`, `abs`, `arksyn`, `kosynus` та допоміжна функція `dovzhyna()` для списків.
- 🐍 Повна струкура інтерпретатора: лексер → парсер → AST → evaluator з середовищем (environment) та накопиченням функцій до старту `main()`.

## Швидкий старт
### Вимоги
- Go 1.26 або вище.

### Запуск прикладу
```bash
go run ./cmd/piton main.piton
```
### Збірка виконавчого файлу
```bash
go build -o piton ./cmd/piton
./piton main.piton
```
Після компіляції можна передавати інші `.piton`-скрипти, але `main()` все одно має викликатися в кінці файлу, щоб започаткувати обчислення.

## Примітиви мови
### Ключові слова
| Українське | Що робить |
|------------|-----------|
| `functia` | оголошення функції без параметрів |
| `nekhay` | оголошення локальної або глобальної змінної |
| `drukuvaty` | вивід рядків/чисел/списків у консоль |
| `vvid` | читання числа з консолі |
| `yaksho` / `inackshe yaksho` / `inackshe` | блочні умови з індентацією |
| `poky ... kinets` | цикл-умова (while) |
| `vernuty` | повернення значення з функції |
| `dovzhyna` | вбудована функція для списків |
| `korin`, `loh10`, `abs`, `arksyn`, `kosynus`, `stupin` | слухняні математичні оператори |

### Контрольні конструкції й стиль
- Блоки коду утворюються лише через відступи (пробіли або таби). Синтаксис наближений до Python, але український.
- `yaksho`, `inackshe`/`inackshe yaksho` та `poky ... kinets` завжди завершуються `:`; без `:` парсер викликає дружню помилку.
- `functia` не підтримує параметри, тому доступ до даних відбувається через глобальні змінні або локальні `nekhay`, які існують в обсязі функційного середовища.
- `main()` викликається в кінці файлу, щоб інтерпретатор міг зібрати всі функції до моменту виконання.

### Типи значень
- **Числа:** `1`, `3.14`, `0.5` — працюють зі звичними арифметичними операторами `+`, `-`, `*`, `/`, `>` і `<`.
- **Рядки:** подвійні лапки (`"текст"`), їх можна друкувати через `drukuvaty`.
- **Списки:** квадратні дужки, індексація `[index]`, довжина через `dovzhyna()`. Індекси завжди приводяться до `int`.

### Математичні та список-фокусні функції
- **`stupin`**: `2 stupin 3` → `8`.
- **`korin`, `loh10`, `abs`, `arksyn`, `kosynus`** використовуються як префіксні оператори: `korin 9`, `loh10 (100)`, `abs (0 - 5)`.
- **`dovzhyna(список)`**: повертає кількість елементів (float). Працює тільки зі списками.
- **Трикутники, логарифми, модуля**: доступні одразу без імпорту.

### Обробка помилок
Інтерпретатор викликає `SyntaxError` з дружнім повідомленням `Ryadok [N]: Ya tut interpretator...` у випадку, якщо зустрічається невідома лексема, неправильний індент або математичне/типове порушення (наприклад, `stupin` між нечислами або ділення на нуль).

## Демонстраційний скрипт `main.piton`
`main.piton` — це повноцінна програма, що демонструє переваги Piton:
- Ітерації над масивом значень зі змінною, яка передається до функції класифікації через глобальний контекст.
- Вбудовані математичні оператори (`stupin`, `korin`, `loh10`, `abs`, `arksyn`, `kosynus`).
- Робота зі списками, агрегатами, індексацією та `dovzhyna()`.
- Цикли `poky`, умовні гілки `yaksho` / `inackshe`, та `vernuty` із допоміжних функцій (`klasify`, `average`, `memo`).
- Показано повторну обробку списків, середнє значення, вибірковий перегляд фундаментальних термінів, а також сумісні тригонометричні обчислення.

### Приклад виклику
```piton
functia main():
    nekhay i = 0
    poky i < dovzhyna(values):
        currentValue = values[i]
        drukuvaty korin currentValue
        drukuvaty klasify()
        i = i + 1
    kinets

main()
```

## Архітектура
- `cmd/piton` — CLI: читає файл, передає код у `interpreter.Run`.
- `interpreter` — фасад: викликає лексер, парсер та `evaluator`, керуючи потоком виводу.
- `internal/lexer` — токенізація й розпізнавання індентації.
- `internal/parser` — побудова AST, визначення довжини виразів та обробка блоків.
- `internal/ast` — структура вузлів програми, виразів і операторів.
- `internal/evaluator` — виконання AST із середовищами, підтримка функцій, циклів, списків.
- `internal/token` — словник ключових слів і лексем.
